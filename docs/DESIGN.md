## 二、会计分录异常检测

### 2.1 核心概念

#### 2.1.1 凭证唯一值
```
格式：{年份}年{月份}月-{原凭证号}
示例：2025年9月-记-0497

数学表达：
VoucherID = Year(Date) & "年" & Month(Date) & "月-" & OriginalNo
```

#### 2.1.2 会计分录特征
```
定义：凭证涉及的所有一级科目，去重后按"、"连接

示例凭证：
  借：应收账款 100
  借：应交税费 20
  贷：主营业务收入 100
  贷：应交税费-销项税 20

会计分录特征：应收账款、应交税费、主营业务收入

数学表达：
Feature(V) = Sort(Unique({Subject | Subject ∈ Voucher V}))
```

#### 2.1.3 对方科目
```
定义：当前分录行对应的反向科目

规则：
- 当前行为借方时，对方科目 = 该凭证所有贷方科目
- 当前行为贷方时，对方科目 = 该凭证所有借方科目

数学表达：
Counter(i) = {Subject | Subject ∈ Voucher V(i), Direction(Subject) ≠ Direction(i)}
```

### 2.3 群距离理论

#### 2.3.1 共现率
```
定义：两个群之间跨界凭证的比例（平滑处理）

公式：
CrossRate(A,B) = (CrossCount(A,B) + 1) / (Count(A) + Count(B) + 2)

其中：
- CrossCount(A,B) = 同时涉及群A和群B的凭证数量
- Count(A) = 归属群A的凭证数量
- +1 和 +2 为拉普拉斯平滑
```

#### 2.3.2 群距离
```
公式：
Distance(A,B) = min(1/x - 1, 100)

其中 x = Σ x_k （对所有包含该群对的k群组合求和）

性质：
- 当跨界凭证多 → CrossRate大 → Distance小（群相近）
- 当跨界凭证少 → CrossRate小 → Distance大（群相远）
- 上限100防止极端值
```

**示例**：
```
总凭证300笔（工资100、销售100、采购100）
- 采购&工资跨界30笔
- 采购&销售跨界15笔
- 采购&销售&工资跨界5笔

x(采购,工资) = (30+1)/(100+100+2) + (5+1)/(100+100+100+2) 
             = 0.154 + 0.020 
             = 0.174

D(采购,工资) = min(1/0.174 - 1, 100) 
             = min(4.75, 100) 
             = 4.75
```

### 2.4 重要性水平理论（新增）

#### 2.4.1 设计目的
剔除小额凭证，聚焦重要业务，减少噪声干扰。

#### 2.4.2 帕累托分析
```
定义：按金额排序，累计占比达到设定百分比的临界点

计算步骤：
1. 对群G内所有凭证按绝对金额降序排序
2. 计算累计金额占比
3. 找到累计占比 = 帕累托百分比 的位置
4. 该位置的金额 = 重要性水平

公式：
Materiality(G) = Amount_k 
其中 k = argmin(|Σ(i=1 to k) Amount_i / Total_Amount - ParetoPercent|)
```

#### 2.4.3 α系数（金额权重）
```
对于凭证V，金额Amount(V)：

IF Amount(V) >= Materiality(Group(V)):
    α(V) = 1.0
ELSE:
    α(V) = Amount(V) / Materiality(Group(V))

用途：α系数用于调整异常得分，小额凭证得分会被降低
```

#### 2.4.4 剔除统计
```
每个业务群独立计算：
- 剔除凭证数 = 金额 < 重要性水平的凭证数量
- 剔除凭证占比 = 剔除凭证数 / 该群总凭证数
- 剔除金额 = 被剔除凭证的金额合计
- 剔除金额占比 = 剔除金额 / 该群总金额
```

### 2.5 异常检测理论

#### 2.5.1 跨模块异常得分
```
对于跨群的凭证V（涉及群G1, G2, ..., Gn）：

Score_cross(V) = Σ Distance(Gi, Gj) + Penalty_other

惩罚机制：
- 跨了其他群（其他群+某群）：+10分
- 完全归入其他群：+20分
```

#### 2.5.2 模块内异常得分
```
对于在群G内的凭证V，其特征为F：

Freq(F) = Count(F in G) / Count(G)
Score_inner(V) = min(1/Freq(F) - 1, 100) + Penalty_other

特殊情况：
- 如果 Freq(F) = 0（从未出现）: Score_inner = 100
- 如果 Freq(F) 极低: Score_inner ≈ 100
- 如果 Freq(F) 正常: Score_inner ≈ 0
```

#### 2.5.3 综合风险得分（含重要性水平）
```
TotalScore(V) = α(V) × (Score_cross(V) + Score_inner(V))

其中 α(V) 是金额权重系数

风险等级：
- TotalScore ≥ 80: 高风险 🔴
- 30 ≤ TotalScore < 80: 中风险 🟠
- 10 ≤ TotalScore < 30: 中低风险 🟡
- TotalScore < 10: 正常 🟢
```

### 2.6 科目连接图理论

#### 2.6.1 图的构建
```
节点：群G内所有出现的科目（去重）

边：贷方科目 → 对方科目（借方科目）

边权重：W(A→B) = |{Voucher | Voucher包含(A贷方, B借方)}|
         即：有多少个凭证包含这条资金流动

重要：权重按凭证计数，不是按行计数！
```

#### 2.6.2 图的特性
```
1. 资金流向：从贷方流向借方（会计恒等式）
2. 不走回头路：单向流动，无环
3. 汇聚性：多个科目可能汇聚到同一科目（多借一贷）
4. 分流性：一个科目可能分流到多个科目（一借多贷）
```

#### 2.6.3 可视化设计
```
布局算法：分层布局（从左到右）
- Layer 0: 源头节点（无入边）
- Layer 1: 中间节点
- Layer 2+: 终点节点（无出边）

视觉设计：
- 节点颜色：按科目类型（资产/负债/费用/成本/收入）
- 节点大小：固定（文本自适应）
- 边线粗细：按权重对数映射
- 数字徽章：权重≥5的边显示数值
- 贝塞尔曲线：柔和曲线连接
```

## 三、对方科目分析

### 3.1 核心概念

#### 3.1.1 分录结构分类
```
系统根据借贷方科目的种类数量，将凭证分为两类处理：

【简单分录】- 直接计算对方科目
1. 1借1贷型
   特征：借方科目种类数 = 1 且 贷方科目种类数 = 1
   处理：直接匹配
   
2. 1借m贷型（m > 1）
   特征：借方科目种类数 = 1 且 贷方科目种类数 > 1
   处理：一借对多贷，虚拟行重构
   
3. m借1贷型（m > 1）
   特征：借方科目种类数 > 1 且 贷方科目种类数 = 1
   处理：多借对一贷，虚拟行重构

【复杂分录】- 穷举求解
4. m借n贷型（m > 1 且 n > 1）
   特征：借方科目种类数 > 1 且 贷方科目种类数 > 1
   处理：使用穷举算法求解所有可行方案

【特殊分录】- 规则拦截
5. 本年利润结转
   特征：包含"本年利润"科目
   处理：对方科目强制填"本年利润"
   
6. 汇兑损益调汇
   特征：包含"财务费用"+多个货币性科目（应收/应付/借款等≥3个）
   处理：非财务费用行的对方科目填"财务费用"

【红字处理】- 金额清洗
全借全贷型（红字冲销）
   特征：借方总额≈0 或 贷方总额≈0（红字分录）
   处理：负数搬家 - 将负数金额移到对侧方向
```

#### 3.1.2 数据预处理
```
1. 生成唯一凭证标识
   _uid = 日期 + "_" + 凭证号

2. 金额清洗（处理红字）
   IF 借方总额 > 0 AND 贷方总额 ≈ 0:
       将负的借方金额移到贷方（取绝对值）
   IF 贷方总额 > 0 AND 借方总额 ≈ 0:
       将负的贷方金额移到借方（取绝对值）

3. 科目名称规范化
   - 去除前后空格
   - 保留原始大小写
```

#### 3.1.3 聚类（复杂分录分组）
```
对于复杂分录（m借n贷），按科目组合模式聚类：

模式Key = MD5(所有科目名称排序后连接)

目的：
- 相同模式的凭证共享解决方案
- 减少重复计算
- 支持批量方案选择

示例：
凭证A：借-原材料/应交税费，贷-应付账款/银行存款
凭证B：借-原材料/应交税费，贷-应付账款/银行存款
→ 属于同一模式，使用相同方案
```

### 3.2 穷举算法（ExhaustiveSolver）

#### 3.2.1 核心思想
```
使用DFS（深度优先搜索）穷举所有可能的金额分配方案。

智能转置优化：
IF 贷方数量 > 借方数量:
    转置计算（将贷方作为drivers）
目的：减少搜索树的分支因子

双轨计算：
1. 宽松模式（use_perfect_lock=False）：快速求解
2. 严格模式（use_perfect_lock=True）：寻找完美匹配
```

#### 3.2.2 关键概念

**敏感科目（Sensitive Subjects）**：
```python
SENSITIVE_KEYWORDS = ["银行", "现金", "Bank", "Cash", "支付宝", "微信"]

约束：敏感科目不允许异号金额混合分配
原因：防止将正的银行存款分配到负的应付账款上（不符合业务逻辑）
```

**Driver vs Bucket**：
```
Driver（驱动方）：数量少的一方，作为外层循环
Bucket（接收方）：数量多的一方，作为内层分配目标

示例：2借方 × 3贷方
- Driver：借方（数量少）
- Bucket：贷方（数量多）
```

**完美锁定（Perfect Lock）**：
```
在严格模式下，先寻找金额完全相等的科目对：
IF 借方科目金额 == 贷方科目金额:
    直接锁定一对一映射
    从后续计算中移除

目的：减少搜索空间，提高求解效率
```

#### 3.2.3 分配策略

**策略A：全匹配（Full Match）**
```
对于Driver科目D（金额d），从Buckets中选择子集：
- 子集金额之和 ≈ d
- 不拆分任何Bucket科目

示例：
Driver: 原材料 1000
Buckets: 应付账款 600, 银行存款 400, 税费 100
→ 选择 {应付账款, 银行存款}，和=1000 ✓
```

**策略B：部分匹配（Partial Match）**
```
对于Driver科目D（金额d）：
1. 选择部分Buckets全取
2. 剩余金额从一个Bucket中部分提取

约束：
- 敏感科目的部分提取有符号约束
- 超额分配检查：|needed| > |target| * 10 时拒绝
  （防止小额目标产生大额回调）

示例：
Driver: 原材料 1000
Buckets: 应付账款 800, 银行存款 500
→ 全取应付账款 800 + 从银行存款取 200
```

#### 3.2.4 DFS搜索流程
```python
def dfs(driver_index, current_allocations, remaining_buckets):
    IF driver_index == len(drivers):
        IF remaining_buckets金额之和 ≈ 0:
            找到一个有效方案
        RETURN
    
    driver = drivers[driver_index]
    possible_splits = generate_combinations(driver, remaining_buckets)
    
    FOR each split IN possible_splits:
        更新current_allocations
        更新remaining_buckets（减去已分配金额）
        dfs(driver_index + 1, ...)
```

#### 3.2.5 结果后处理
```
1. 转置还原：如果是转置计算，还原结果方向
2. 金额聚合：基于完整Key聚合（保留正负后缀）
3. 会计守恒校验：
   - 借方分配金额之和 = 原始借方金额
   - 贷方接收金额之和 = 原始贷方金额
4. 签名去重：MD5哈希去重相同方案
5. 限制返回数量：最多max_solutions个（默认200）
```

### 3.3 奥卡姆剃刀评分（OccamsRazor）

#### 3.3.1 评分公式
```
Score = 100 - (行数惩罚) - (分拆惩罚 × 硬骨头系数)

其中：
- 行数 = 借方科目数 + 贷方科目数
- 分拆惩罚根据Driver/Bucket角色不同
```

#### 3.3.2 硬骨头科目
```python
HARD_BONES = ["应交税费"]

硬骨头系数 = 2.0（普通科目为1.0）

原理：某些科目（如应交税费）业务逻辑简单，不应被随意拆分
```

#### 3.3.3 分拆惩罚计算
```
IF 借方是Driver（n >= m）:
    借方分拆惩罚 = (分拆次数 - 1) × 5.0 × 硬骨头系数
    贷方分拆惩罚 = (分拆次数 - 1) × 1.0 × 硬骨头系数
ELSE（贷方是Driver）:
    贷方分拆惩罚 = (分拆次数 - 1) × 5.0 × 硬骨头系数
    借方分拆惩罚 = (分拆次数 - 1) × 1.0 × 硬骨头系数

说明：
- Driver（数量多的一方）分拆惩罚重（5分/次）
- Bucket（数量少的一方）分拆惩罚轻（1分/次）
```

#### 3.3.4 方案排序与选择
```
1. 计算所有方案的奥卡姆得分
2. 按分数降序排列
3. 返回Top-K方案（默认最多200个中的前几个）
4. 用户可手动选择非最高分方案
5. 同模式凭证共享用户选择
```

### 3.4 数据压缩模式

#### 3.4.1 精简压缩（Simple）
```
适用场景：几万到几十万行数据

逻辑：
1. 按（凭证+科目+摘要）分组
2. 分别累加借方金额和贷方金额
3. 关键：同一科目既有借又有贷时，拆成两行

保留字段：制单日期、凭证编号、一级科目、科目明细、借方金额、贷方金额、摘要
```

#### 3.4.2 极简压缩（Minimal）
```
适用场景：50万行以上的超大型序时账

逻辑：
1. 提取月份
2. 生成分录特征（按原始凭证聚合）
3. 按（月份+分录特征+一级科目）聚合
4. 生成虚拟凭证号

目的：将多个月份的相同业务模式的凭证合并，大幅减少数据量
```

### 3.5 结果重构与导出

#### 3.5.1 简单分录重构
```
1借1贷：
- 借方行：对方科目 = 贷方科目
- 贷方行：对方科目 = 借方科目

1借m贷/m借1贷：
- 原始行：保留原行，对方科目填单一侧科目
- 虚拟行：为单一侧科目生成虚拟行，对方科目分别填多侧科目
```

#### 3.5.2 复杂分录重构
```
基于选定的方案（最佳或用户选择）：

借方重构：
FOR each 借方科目:
    FOR each 分配的贷方科目:
        按金额比例拆分原始行
        生成新行，对方科目 = 贷方科目

贷方重构：
FOR each 贷方科目:
    FOR each 分配的借方科目:
        按金额比例拆分原始行
        生成新行，对方科目 = 借方科目
```

#### 3.5.3 列重排导出
```
输出列顺序：
原始列... + ["对方科目"] + 原始列后续...

对方科目列插入在"贷方金额"列之后
```

### 1.我们需要如下的信息

- **凭证号唯一值**：原先的会计凭证里的凭证号，通常都只有一个"记-xx"，在多个月里，会有重复值，我们按记账日期，给每个凭证号生成一个唯一值，具体如下：
$$
year(记账日期)&"年"&month(记账日期)&"月-"&凭证原编号
$$
效果如下：
记-0497 --> 2025年9月-记-0497

- **对方科目**：我们需要每一行会计科目的对方科目

- **会计分录特征**：我们需要每一个凭证的会计分录特征

### 2. 会计分录特征与其特性

- 2.1 什么是会计分录特征：

什么是会计分录特征？我给你你举个例子，这是两张会计凭证：

凭证    会计科目       明细科目     借方金额     贷方金额
001       固定资产        A客户                                xx   
001       其他应收款      个人           xx                    
002       其他应收款      B客户                              xx 
002       管理费用                            xx                  

凭证001的会计分录特征：固定资产、其他应收款
凭证002的会计分录特征：其他应收款、管理费用

会计分录特征非常简单，它就是对一级科目进行了聚合，变成了1行，并且通过“、”分割

-2.2 会计分录特征的特性：一惯性

因为所有凭证都有一级科目，所以所有凭证都有其会计分录特征。

同个会计分录特征，它们所表达的业务意思，往往是一致的。比如，下面这两张凭证：

凭证    会计科目       明细科目     借方金额     贷方金额
001       应收账款        A客户             xx                      
001       营业收入       材料                                 xx       
002       应收账款      B客户              xx                          
002       营业收入      材料                               xx       

这就可以变为“应收账款、营业收入”，在这个会计分录特征里，通常每个凭证都是一个意思，都是卖东西

一份序时账，可以划分为多个会计分录特征

### 3. 需要的信息汇总

我们需要如下信息：

-**自带型**：[记账日期，凭证编号，一级科目，对方科目，借方金额，贷方金额]

-**需要再加工型**：[凭证编号（唯一值版）、会计分录特征]


## 二、我们怎么识别异常？

### 1.哪些是异常点

首先，我们的会计分录测试，必须摒弃那些教条主义的东西。

比如说，在一些会计分录测试的理论上，会关注记账人有没有变动、有没有经过审核、记账日期是不是在周末等等

我认为这些是无意义的，在这一轮我仍坚持我的判断，因为根据我的实务经验，95%的企业，在这两个地方都是随意设置得，比如记账人可能就是个admin，或者时间不会带上分时秒，只有日期，而且都是月末最后一天。
即便时间、记账人有变动，我觉得这也不能代表什么。诚然，现在的会计分录测试理论中，是有提出测试这两个地方，但我认为这明显只是完全地照搬西方理论，一个拙劣地模仿罢了，完全不符合中国实际。

还有就是，摘要是有用的，但是因为它实在是太不可控，太乱了，**所以我们也抛弃掉摘要。**

我们的异常点只有：**会计科目组合、金额**。

这两个是最干净、最普适的。

### 2. 什么叫异常

我们已经确定，我们只通过会计科目、金额来找异常点，我们判定哪些是异常的情况：

会计科目组合，比如说有：“借应收账款、借：应付账款、贷：主营业务收入”的会计分录，这种就是组合异常的。


## 三、识别异常的SOP（重要）

我们主要基于**会计分录特征***来识别异常。

### Step1 人工设定借贷锚定对，将会计分录特征进行群聚类

我们可以把一份序时账，看为很多个会计分录特征，它可能会是这样的：

| 会计分录特征 |
| :--- |
| 半成品、委托加工物资 |
| 半成品、原材料、存货跌价准备 |
| 半成品、原材料、管理费用 |
| 半成品、原材料、委托加工物资、应付账款暂估 |
| 半成品、原材料、委托加工物资、应付账款暂估、材料采购过渡 |
...

我们预设一些核心词，比如
```python
停用词库 = ["银行存款", "库存现金", "应交税费"]
采购核心群 = ["原材料", "半成品", "库存商品","委托加工物资", "应付账款", "预付账款"]
销售核心群 = ["应收账款", "应收票据", "主营业务收入", "其他业务收入","发出商品","主营业务成本"]
薪酬核心群 = ["应付职工薪酬"]
生产核心群...
长期资产核心群..
```

比如上面的会计分录特征，它会被这样划分
| 会计分录特征 |业务活动群 |
| :--- |:--- |
| 半成品、委托加工物资 |采购活动群|
| 半成品、原材料、存货跌价准备 |采购活动群|
| 半成品、原材料、管理费用 |采购活动群|
| 半成品、原材料、委托加工物资、应付账款暂估 |采购活动群|
| 半成品、原材料、委托加工物资、应付账款暂估、材料采购过渡 |采购活动群|
...

我的群划分逻辑：

- 一借一贷型：借贷数量2个，2个科目对象都在一个群，归入该群；2个科目对象在不同的群，各自归入其他群

- n借m贷型：借贷数量n+m个，至少2个科目对象都在一个群，归入该群；至少2个科目对象在多个群，各自归入其他群

在上面的示例里，都是n借m贷型的会计分录，它们至少有2个科目对象都在采购核心群里，所以它们都被划分到了采购活动群。

同时，针对群判断逻辑，我们打了个补丁，我打的补丁就是：停用词

比如，一份凭证是 借：管理费用 贷：银行存款。

这是个一借一贷，根据规则，有一个科目对象在群里就行，但是，银行存款这个对象，在销售、采购里都有，系统一看：“哎呀，银行存款在采购群里，也在销售群里！” 结果它可能会把这笔报销，强行跟采购或销售扯上关系。

所以，**对于重复出现在群里的科目对象，我们必须停用，保证每个群的科目对象都是唯一的**

### Step2 机器归类

根据我们已经设定好的聚类词，已经可以聚类大部分会计分录特征了，少部分无法聚类的，交给机器来判断。

无法聚类的情况，我分为2种：

- 一借一贷型：两个科目对象都不在群聚类对象里面，比如：

借：委托加工物资
贷：银行存款

由机器进行词向量空间判断，如果委托加工物资，机器判断为≈原材料，那就归入到采购群中。

但如果是像这种，机器都无法判断的：

借：预计负债
贷：营业外支出

直接归入其他群，对，我们还要设计一个其他群。

同时，这里的机器学习应该是一次性的，而不是累计的，也就是每次运行时机器都重新学一次，而不是一直累计学习。

因为，不同序时账的拓扑结构可能不一样，机器学习在这一份序时账的经验无法应用于另一份。


### Step3 群距离计算

这里，我提出一个更简单、好用的群距离算法：

- 共现率：x = 跨界凭证数量 / (群A总凭证数量 + 群B总凭证数量)

同时，为了避免样本量过少，算出来的值失真，我们做一些平滑处理：

$$
x = (跨界凭证数量 + 1) / (群A总凭证数量 + 群B总凭证数量 + 2)
$$

- 群距离：$$D =-ln(x)$$

这个算法非常简单，你试想，如果两个群之间，跨界凭证非常多，那本身就说明了，这两个群就是很近的，所以，x的值就会比较小，群距离就近；反之，群距离就远

这是直接根据实际情况去设计距离；

计算好之后，应该会有一个每个群之间的距离矩阵：

| 群对    | 跨界凭证数 | 总凭证数 | $x$    | 距离 $D$   |
| ----- | ----- | ---- | ------ | -------- |
| 采购-资产 | 50    | 5000 | 0.01   | 4.6 (远)  |
| 采购-税务 | 1200  | 5000 | 0.24   | 1.4 (近)  |
| 工资-资产 | 2     | 3000 | 0.0007 | 7.3 (极远) |
| 销售-税务 | 800   | 4000 | 0.20   | 1.6 (近)  |
|
...

## Step4 异常凭证识别

-1. 跨模块的异常凭证：

我们将它跨过的模块，在群距离矩阵的得分加起来。

$$Score_{cross} = ΣD_{ij}$$

-2. 没跨模块的异常凭证：

比如都在【销售群】里。大部分凭证的特征是 [应收账款, 主营业务收入]。
突然来了一笔 [应收账款, 营业外支出]（强行核销死账），它因为有应收，被分到了销售群，没跨界。

我们直接看这笔特征在该群内部出现的频次（占比）。

如果该群有 10000 笔凭证，这个组合只出现了 2 次，频次为:

frep(F)=2/10000

异常得分为：-ln(y)≈8.5

$$Score_{inner} = -ln(frep(F))$$

-3. 综合风险得分：

$$TotalScore = Score_{cross} + Score_{inner}$$

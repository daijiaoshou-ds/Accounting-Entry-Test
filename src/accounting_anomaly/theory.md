## 前言
什么是会计分录测试？这里我就不引经据典咬文嚼字了，反正，会计分录测试的测试的目的在于，找出异常的会计凭证。

今天，我的这一套会计分录测试理论，将抛弃教条主义，只从最纯粹、最普适的会计科目和金额出发，找出异常的会计凭证。

**主要测试点：会计科目+金额**

像一些记账时间（是否为周末记账、凌晨记账），记账是否经过审核之类的，我将一概不管。毕竟在实操中，有不少凭证都是在月末统一录入的，比如成本结转凭证（月末一次加权平均法核算下），计提工资等等，所以，这一些方面的测试没有多少意义，更多是形式主义。

## 二、理论框架

### 一、异常点查找

什么时候可以断定，某张会计凭证是异常的？会计科目的异常组合，比如这一张会计凭证：

```markdown

借：原材料
贷：预计负债

```

这一张明显就是异常的，因为哪有将原材料和预计负债核算在一起的，对吧？

但是，一份序时账里，有非常多的会计凭证，我们该如何去识别呢？

### 二、会计分录特征

我们首先来完成对序时账的压缩：

- **1.什么是会计分录特征**

我给你举个例子，这是两张会计凭证：

凭证    会计科目       明细科目     借方金额     贷方金额
001       固定资产        A客户                                xx
001       其他应收款      个人           xx
002       其他应收款      B客户                              xx
002       管理费用                            xx

凭证001的会计分录特征：固定资产、其他应收款
凭证002的会计分录特征：其他应收款、管理费用

会计分录特征非常简单，它就是对一级科目进行了聚合，变成了1行，并且通过“、”分割

- **2.会计分录特征的特性：一惯性**

因为所有凭证都有一级科目，所以所有凭证都有其会计分录特征。

同个会计分录特征，它们所表达的业务意思，往往是一致的。比如，下面这两张凭证：

凭证    会计科目       明细科目     借方金额     贷方金额
001       应收账款        A客户             xx
001       营业收入       材料                                 xx
002       应收账款      B客户              xx
002       营业收入      材料                               xx

这就可以变为“应收账款、营业收入”，在这个会计分录特征里，通常每个凭证都是一个意思，都是卖东西

一份序时账，可以划分为多个会计分录特征

| 会计分录特征 |
| :--- |
| 半成品、委托加工物资 |
| 半成品、原材料、存货跌价准备 |
| 半成品、原材料、管理费用 |
| 半成品、原材料、委托加工物资、应付账款暂估 |
| 半成品、原材料、委托加工物资、应付账款暂估、材料采购过渡 |
...

这样，数万张会计凭证，就变成了数百个会计分录特征，一个会计分录特征中可能包含数百笔或者数千笔会计凭证。

这里，我们就完成了对信息的第一次压缩，即便是数万、数十万张的会计凭证，最终它通常都会压缩到几百个会计分录特征。

因为会计分录特征实际是业务特征，业务核算越复杂，会计分录特征越多，大部分公司，会有上万张凭证，但不会有上万个会计分录特征。

### 四、群距离计算

从跨群来看，我们就可以发现端倪了。

比如说：
- 一个会计凭证：分别跨过了采购活动群和销售活动群，这凭证大概率就是有异常的，因为很少有凭证同时确认采购和销售。
- 一个会计凭证：分别跨过了采购互动群和税务活动群，好像又没什么问题，因为确认采购时经常会带上“应交税费”。

所以，我们不能把所有跨群凭证都定义为异常凭证，要看它跨了哪些群，这些群与群之间距离是否相近？

可是，我们作为人类，是知道“采购活动群”和“税务活动群”是相近的，“采购活动群”和“销售活动群”是远的，机器要如何知道呢？而且，我们只知道远近，能定量描述出距离吗？

其实，要定量计算群之间的距离，很简单，算一个除法就知道了：

$$
\frac{跨群凭证数量}{被跨群A的凭证数量 + 被跨群B的凭证数量}
$$

比如说，采购活动群和税务活动群的跨群凭证数量非常多，那它的分子就会很大，除出来的比率比率就会比较小。

基于实际观测，就可以算出每个群之间的距离。

当然，这是基本的计算公式，在实际群距离计算公式中，我基于这个基本计算公式，做了一点数学变体（融入了Noisy-OR模型）：

$$X_{ij} = 1 - \prod_{k=2}^{n} \prod_{S \in C_k} \left(1 - \frac{n_S + 1}{\sum_{u \in S} |M_u| + 2}\right)$$


|         符号         |                                     含义                                     |
| -------------------- | ---------------------------------------------------------------------------- |
| $X_{ij}$             | 群 $i$（如：研发）和群 $j$（如：生产）之间的综合共现率，最终结果，值域 $\in [0,1)$ |
| $\prod$              | 连乘符号，把括号里的东西全乘起来                                                |
| $k$                  | 共现阶数，$k=2$ 表示两个群直接共现，$k=3$ 表示通过第三个群间接共现，以此类推        |
| $C_k$                | 在第 $k$ 阶下，所有同时包含群 $i$ 和群 $j$ 的组合集合                            |
| $S$                  | $C_k$ 里的一个具体组合，比如 $\{研发, 生产, 采购\}$                             |
| $n_S$                | 组合 $S$ 里所有群同时出现在同一张凭证上的凭证数量                                |
| $\|M_u\|$            | 群 $u$ 的凭证总数                                                             |
| $n_S + 1$ 和 $+2$    | 拉普拉斯平滑，防止分子为0导致概率为0                                            |
| $1 - (\cdots)$       | 对立事件，"这个证据没有触发"的概率                                              |
| 最外层的 $1 - \prod$ | 再取对立事件，"所有证据里至少一个触发"的概率                                     |


用一个例子来解释：

```
已知数据：
研发群：25张凭证
生产群：30张凭证
采购群：378张凭证
长期资产群：20张凭证

跨群共现情况：
研发&生产：13张
研发&生产&采购：1张
研发&生产&长期资产：2张
```

**第一步：计算每个组合的 $p_S$**

$$p_{S1} = \frac{13+1}{25+30+2} = \frac{14}{57} = 24.56\% \quad \text{（k=2，研发\&生产）}$$

$$p_{S2} = \frac{1+1}{25+30+378+2} = \frac{2}{435} = 0.46\% \quad \text{（k=3，研发\&生产\&采购）}$$

$$p_{S3} = \frac{2+1}{25+30+20+2} = \frac{3}{77} = 3.90\% \quad \text{（k=3，研发\&生产\&资产）}$$

**第二步：计算每个 $p_S$ 的对立面 $(1 - p_S)$**

$$1 - p_{S1} = 1 - 24.56\% = 75.44\%$$

$$1 - p_{S2} = 1 - 0.46\% = 99.54\%$$

$$1 - p_{S3} = 1 - 3.90\% = 96.10\%$$

**第三步：连乘所有对立面**

$$\prod(1-p_S) = 75.44\% \times 99.54\% \times 96.10\% = 72.10\%$$

> 这个 $72.10\%$ 的含义是：**所有证据全部哑火、研发和生产完全没有共现关系**的概率

**第四步：取反，得到 $X_{ij}$**

$$X_{ij} = 1 - 72.10\% = 27.90\%$$

> 这个 $27.90\%$ 的含义是：**综合所有阶的证据，研发和生产存在共现关系**的概率

得到共现率$X$之后，我们对共现率求倒数，得到距离$D$

$$
D=min(\frac{1}{X}_{ij} -1 , 100)
$$

在上面的例子中：

$$D = \min\left(\frac{1}{27.90\%} - 1,\ 100\right) = \min(2.58,\ 100) = 2.58$$

### 五、群内异常检查

上面的一套组合拳，针对的是跨群的会计凭证。如果是没跨群的会计凭证，是不是就相安无事了呢？

并不是，没跨群的会计凭证，依旧需要计算异常得分。我们要怎么计算异常得分呢？

依照它的罕见程度。在上文中，我们有每个业务活动群之间的科目连接图：

比如说，采购群，它的会计分录连接情况，可能是：

应付账款--100-->原材料

银行存款--100-->应付账款

原材料--50-->委托加工物资

原材料--1-->预计负债

共有251个连接

其中，箭头的数字表述凭证数量，“应付账款--100-->原材料”表示，有100笔凭证的做法是“借：原材料，贷：应付账款”，流向就是表示贷方流向借方。

现在，我们来看一下整个采购群里，每个连接的异常得分：

我们检测的异常点就在于：大家都在往这个科目走，为啥你就偏偏不一样？

这就是群内异常得分的检测方向，就是找罕见的。

这里，我们依旧要上数学模型，我命名为**HHI自适应惩罚模型**，模型总共分为几步走。

- **1. 先看采购群的集中度**

计算HHI

HHI是用来衡量市场集中度的一个指标，我们引入HHI，来计算每个群的连接集中度。
$$
HHI = \sum_{i=1}^{m} (f_i)^2 = \sum_{i=1}^{m} \left( \frac{n_i}{N} \right)^2
$$

如果是集中度比较高的群，因为长尾效应的存在，我们对一些稀疏连接会更加宽容，避免错杀一些连接数少但正常的会计分录；如果是集中度比较低的群，我们会增加对稀疏连接的惩罚，因为这说明这个群很不可控。

- **2. 计算连接频次f**

对于待评分的特定连接，计算其在群组内的相对频次\$f\$：

$$
f = \frac{n}{N}, \quad f \in (0, 1]
$$

- **3.计算异常得分 ($Score$)**

将群组集中度 \$HHI\$ 作为自适应参数，代入惩罚函数计算得分。该函数通过指数 \$-(1-HHI)\$ 动态调节惩罚力度。

$$
\begin{aligned}
S(f) &= \begin{cases}
\dfrac{f^{-(1-HHI)} - 1}{1-HHI}, & HHI \in [0, 1) \\[1.5em]
-\ln f, & HHI = 1
\end{cases} \\[1em]
\text{Scoreinner}(f) &= \min\bigl(100,\ S(f)\bigr)
\end{aligned}
$$

这一套公式真的非常优雅，它来源于Tsallis熵计算公式：

**Tsallis 熵**
$$
Score = \frac{x^{-(q-1)} - 1}{q-1}
$$

我们直接令：**$q = 2 - HHI$**，得到：

$$Score = \frac{x^{-(1-HHI)} - 1}{1-HHI}$$
它的优雅之处在于哪里？

*   **分散群场景 ($HHI \to 0$)**：
    *   **业务含义**：群组内分录杂乱，缺乏主流业务模式。此时模型对稀疏连接“零容忍”，退化为强力倒数惩罚。
    *   **数学极限**：
        $$
        \lim_{HHI\to 0} \text{Score}(f) = \frac{1}{f} - 1
        $$

*   **集中群场景 ($HHI \to 1$)**：
    *   **业务含义**：群组内业务高度统一（如大部分都是“借贷：应付-原材料”）。根据长尾效应，此时模型判定稀疏连接为特殊正常事项的概率较高，退化为温和对数惩罚。
    *   **数学极限**：
        $$
        \lim_{HHI\to 1} \text{Score}(f) = -\ln f
        $$
 啥意思，我结合审计的思路说一下你就明白了！
 
 **HHI像是在测试内控：**
 
 - 如果HHI很高，这个群大部分都集中于某几个分录，说明问题比较小，我们对它的连接频次计算会更宽容，计算出来的异常得分就会比较低！
 
- 如果HHI很低，说明这个群很稀疏，做的账乱七八糟，我们会对严格计算它的连接频次，计算出来的异常得分会偏高！

到这里，我们已经计算出了两个得分，分别是跨群得分Scorecross和群内异常得分Scoreinner。

这两个得分你可以注意到，量纲是不统一的：
- Scorecross：倒数计算，数值偏大
- Scoreinner：倒数/对数计算，数值可能大可能小

我的逻辑在于：cross占得分的大头，inner相较于cross，已经没有那么异常，所以它的得分会稍微低一点。而且，如果inner内部实在太异常，HHI就会变成同样的倒数计算，接近cross的量纲，如果是集中性的（集中性异常比较小），它会变成对数计算，只是给cross的分数做锦上添花。

### 六、重要性水平

这里的重要性水平与审计里“重要性水平”相差甚远，这里的重要性水平更多是信息论中“重要性水平”，即重要信息的意思。

我们基于以下几步来设计重要性水平

- **1.金额绝对值化**

我们以借方金额筛选，并把所有金额绝对值化，避免遗漏掉了负数的会计分录

比如，如果我们设置了个10万元重要性水平，遇到这种会计凭证：

借：原材料    10万
借：应交税费   -2万
贷：应付账款     8万

这个凭证的借方合计只有8万，在不进行全部绝对值化的话，就会错过这个凭证。

全科目绝对化后：

借：原材料    10万
借：应交税费   2万
贷：应付账款     8万

借方金额12万大于10万，就可以被筛选到。
这时候不管它借贷是否平衡，反正这只是我们临时筛选用的。

- **2.分群设计重要性水平**

我们在前面已经完成了群聚类，所以可以利用群来设计不同的重要性水平。

比如说，资产群都是些大额的，采购群可能相较小额，如果我们设了个统一的重要性水平，可能就会厚此薄彼了。

我们运行**帕累托累计**的思路来分群设计重要性水平：

1.在每个独立的业务群内，把凭证按金额从大到小排序。
2.开始累加金额，直到累加的金额达到了该群总交易额的 95%。
3.此时碰到的最后一张凭证的金额，就是这个群的动态重要性水平

- **3.α系数**

如果低于重要性水平，就直接过滤掉，完全不管吗？

如果设置的重要性水平是10万元，有一张凭证的总额是9.9万元，就完全不管，那是怪可惜的。

针对这个情况，我设计出了一个α系数，它也是一个分段函数:

$$
\alpha = 
\begin{cases} 
1, & Amount \ge M_{base} \\
e^{k \cdot \left( \frac{Amount}{M_{base}} - 1 \right)}, & Amount < M_{base}
\end{cases}
$$


α系数其实是个衰减系数，它的逻辑很简单：

- 如果凭证总额大于重要性水平，就为1，不衰减，原汁原味保留
- 如果凭证总额小于重要性水平，就自适应地衰减，也不强行让它风险直接等于0

另外，在下半段公式中，有一个参数k，参数k默认=**5.36**

参数k=5.36是怎么来的呢？

我们设定： "当金额达到重要性水平的P%时，异常分数至少保留Q%"

比如我们认为：当金额达到70%时，分数至少保留20%

的公式是：

$$\alpha = e^{k \cdot \left(\frac{Amount}{M_{base}} - 1\right)}$$

把锚点代入，$\alpha = Q = 0.2$，$\frac{Amount}{M_{base}} = P = 0.7$：

$$0.2 = e^{k \times (0.7 - 1)} = e^{-0.3k}$$

两边取自然对数：

$$\ln(0.2) = -0.3k$$

$$-1.609 = -0.3k$$

$$k = \frac{1.609}{0.3} \approx 5.36$$

在k=5.36下，α系数的衰减效果如下：

| 金额占比 |                 计算过程                 |     α     |        含义         |
| -------- | --------------------------------------- | --------- | ------------------ |
| 99%      | $e^{5.36 \times (0.99-1)} = e^{-0.054}$ | **94.7%** | 卡线造假，几乎不打折 |
| 70%      | $e^{5.36 \times (0.7-1)} = e^{-1.61}$   | **20%**   | 我们设定的锚点       |
| 50%      | $e^{5.36 \times (0.5-1)} = e^{-2.68}$   | **6.9%**  | 半程金额，大幅压制   |
| 30%      | $e^{5.36 \times (0.3-1)} = e^{-3.75}$   | **2.4%**  | 小额，基本消失       |

也就是说，如果重要性水平为10万元，如果有一张异常凭证的金额是9.9万元，依旧会有很高的异常得分，不会被直接忽略掉。

而如果凭证总额已经到了3万元，它的异常得分会被大幅衰减，只有原来得分的2.4%。

对于α系数的运用没看明白？下面我们实际来算一下就知道了。


### 七、异常得分评价

现在万事俱备，只欠东风，我们在前面已经完成了所有计算：
- Scorecorss：跨群凭证的异常得分
- Scroeinner：群内异常得分
- α系数：衰减系数

最终的得分公式为：
$$
TotalScore = (Scorecross + Scoreinner) \times \alpha
$$

在综合得分计算中，α系数的运用：
假设动态重要性水平 $M_{base} = 100,000$ 元，$k = 5.36$。

**场景 1：完全压线或超标（Amount = 100,000 或 150,000）**
*   直接命中上半段公式：**$\alpha = 1$ （100% 分数全额保留）**

**场景 2：卡线（Amount = 99,000 元，也就是 99%）**
*   命中下半段公式：金额略小于阈值。
*   指数部分：$5.36 \times (0.99 - 1) = -0.0536$
*   计算 $\alpha = e^{-0.1} \approx 0.9478$
*   *系统判定：异常分数依然保留 **94.78%**！由于本身 Cross/Inner 异常分很高，依然在榜单前列！*

**场景 3：常规的擦边球（Amount = 80,000 元，也就是 80%）**
*   命中下半段公式。
*   指数部分：$5.36 \times (0.8 - 1) = -1.072$
*   计算 $\alpha = e^{-1.072} \approx 0.3424$
*   *系统判定：虽然也异常，但金额离重要性水平还有点距离，给保留 **34.24%** 的分数。*

**场景 4：无关紧要的垃圾凭证（Amount = 30,000 元，也就是 30%）**
*   命中下半段公式。
*   指数部分：$5.36 \times (0.3 - 1) = -3.752$
*   计算 $\alpha = e^{-3.752} \approx 0.0234$
*   *系统判定：金额太小，分数压缩至 **2.34%**，基本放过*

最后的评分设计：
```python
    def _get_risk_level(self, score):
        """风险等级判断"""
        if score >= 80:
            return '高风险 🔴'
        elif score >= 30:
            return '中风险 🟠'
        elif score >= 10:
            return '中低风险 🟡'
        else:
            return '正常 🟢'
```
